## 9. Environment fallback

Rust toolchain unavailable in a validation environment is a validation-environment limitation, not a ticket pass. The static script only proves package structure. A Rust-enabled Ticket Implementation or QA Session must still run the cargo checks before `TKT-0001` can become Done.

## 10. Step15 Rust preflight requirement

`TKT-0001` cannot be accepted from `scripts/check_rust_workspace_static.py` alone. A Rust-enabled session must run:

```bash
scripts/run_rust_preflight.sh --scope current-package --out reports/preflight/latest
scripts/check_runtime_evidence_files.py --path reports/preflight/latest/rust_preflight_report.json --require-scope current-package --require-pass
```

A `block` verdict means the Rust-enabled CI or Codex Cloud environment must be repaired. A `reject` verdict means the failed command logs must route through the failure feedback route path before parser or gameplay tickets proceed.


## 11. Step16 environment requirement

Before `TKT-0001` can claim runtime evidence, the runner must satisfy `docs/86_codex_cloud_environment_setup.md` and `docs/87_secret_and_network_policy.md`.

The session must record one of these setup evidence paths:

```bash

```

A failure in setup, pinned toolchain installation, or environment evidence validation is a `block` verdict until the Codex Cloud / CI environment is repaired. It is not a Rust workspace implementation reject unless cargo or `taiko_cli` runs and fails after setup succeeds.
