# Ticket Transition Reports

Status: canonical

This directory stores loop-controller ticket advance evidence generated by `scripts/loop_advance_ticket.py`.

Expected files:

```text
reports/loop/ticket_transitions/<run_id>.json
reports/loop/ticket_transitions/<run_id>.md
```

These reports are generated from `reports/loop/merge_history/<run_id>.json` after a merge. They record which ticket moved to `Done`, which dependency-satisfied ticket became `Ready`, and whether the transition was `pass`, `block`, or `reject`.

GitHub Actions uses these reports only as deterministic controller evidence. It does not call Codex/GPT and does not require `OPENAI_API_KEY` or `CODEX_API_KEY`.
