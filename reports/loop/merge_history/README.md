# Merge History Reports

Status: canonical

The Step21 loop controller writes one JSON file per autonomous merge:

```text
reports/loop/merge_history/<run_id>.json
```

These records are evidence for ticket advancement and for automated revert selection. They are normally generated by `scripts/loop_auto_merge_pr.sh` or `.github/workflows/loop-controller.yml`.

A record must include:

- `run_id`
- `ticket_id`
- `pr_number`
- `merge_method`
- `base_branch`
- `head_sha`
- `metadata_path`
- `qa_verdict_path`
- `controller_workflow`
- `merged_at_utc`
